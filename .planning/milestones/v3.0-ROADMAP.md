# Milestone v3.0: AI — Orçamento, Notificações, Deploy e Assistente Conversacional

**Status:** ✅ SHIPPED 2026-05-07
**Phases:** 5-7
**Total Plans:** 6 (05-01, 05-02, 06-01, 06-02, 07-01)

## Overview

Consolidação do MVP com orçamento e metas de gasto (BUDG), notificações por email (NOTF), infraestrutura de produção com PostgreSQL, CI/CD e deploy automatizado (INFR), e assistente financeiro conversacional com IA (IA-01/02/03).

## Phases

### Phase 5: Orçamento e Metas

**Goal:** Usuários podem definir metas de gasto por categoria e acompanhar progresso no dashboard.
**Depends on:** v1.0 (all phases)
**Plans:** 2 plans

Plans:
- [x] 05-01: Backend MetaGasto (modelo, endpoints CRUD, campos computados)
- [x] 05-02: Frontend Budget UI (BudgetView, BudgetEditModal, dashboard integration)

**Details:**
- Modelo `MetaGasto` (categoria, valor meta, mês/ano, usuário)
- Endpoint CRUD `/api/metas/` com campos computados (gasto_realizado, percentual, status)
- Dashboard integration com progress bars color-coded
- Alerta visual quando gasto ultrapassa 80% da meta (toast + dashboard insights)
- Alerta crítico quando > 100%

### Phase 6: Notificações e Deploy

**Goal:** Notificações push/email e infraestrutura de produção (PostgreSQL, CI/CD, deploy).
**Depends on:** Phase 5
**Plans:** 2 plans

Plans:
- [x] 06-01: Backend notificações e jobs (Celery/Redis, SendGrid, cron-job.org)
- [x] 06-02: Infraestrutura (PostgreSQL, GitHub Actions CI/CD, Railway + Vercel)

**Details:**
- Tasks `send_weekly_reminder` e `check_monthly_average` (api/tasks.py)
- Endpoint `/api/tasks/trigger/` protegido por secret
- SendGrid SMTP configurado
- PostgreSQL via DATABASE_URL (Railway add-on)
- CI/CD GitHub Actions: check + deploy backend + deploy frontend
- Backend Railway: campo-valor-production.up.railway.app
- Frontend Vercel: sem-aperreio.vercel.app

### Phase 7: IA-01 Assistente Financeiro Conversacional

**Goal:** Criar um assistente financeiro conversacional que interpreta linguagem natural para registrar gastos e receitas.
**Depends on:** v2.0 (all phases)
**Plans:** 1 plan

Plans:
- [x] 07-01: Assistente IA parser + drawer chat + FAB

**Details:**
- Backend: endpoint `POST /api/ai/chat/` com parser estruturado (OpenAI GPT-4o-mini + fallback)
- Frontend: componente AIAssistant.vue (drawer lateral/chat)
- IA-02: contextual multi-etapas com perguntas complementares
- IA-03: continuidade conversacional, saudações, fim de conversa amigável

## Milestone Summary

**Key Decisions:**
- Fallback para endpoint + cron-job.org quando Celery worker não roda no free tier
- SQLite local / PostgreSQL produção via DATABASE_URL
- Options API no Vue mantida (migração Composition API candidata para v2)

**Issues Resolved:**
- DATABASE_URL não lido (python-decouple lê .env primeiro) → usar os.environ.get()
- CORS bloqueado → fallback para FRONTEND_URL
- Vercel MIME type error → adicionar "handle": "filesystem"
- Cron jobs 404 → URLs atualizadas para domínio production
- Task error 500 → corrigir related_name gastos__

**Issues Deferred:**
- Celery Beat interno (depende de upgrade Railway)
- Testes automatizados

**Technical Debt Incurred:**
- Modelo ML treinado on-demand (não persistido)
- Paginação simples (apenas limite, sem metadados completos)
- Sem testes automatizados
