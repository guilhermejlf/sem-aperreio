## Wave 2: GitHub Actions CI/CD

### Task 2.1 - Workflow de Deploy

**Files modified:** `.github/workflows/deploy.yml` (new)

**Action:**

1. Criar `.github/workflows/deploy.yml`:
   ```yaml
   name: Deploy

   on:
     push:
       branches: [main]

   jobs:
     check:
       name: Check
       runs-on: ubuntu-latest
       steps:
         - uses: actions/checkout@v4
         
         - name: Setup Python
           uses: actions/setup-python@v5
           with:
             python-version: '3.11'
             cache: 'pip'
         
         - name: Install Python deps
           run: |
             pip install -r requirements.txt
         
         - name: Django Check
           run: |
             python manage.py check
           working-directory: .
         
         - name: Setup Node
           uses: actions/setup-node@v4
           with:
             node-version: '20'
             cache: 'npm'
             cache-dependency-path: frontend/package-lock.json
         
         - name: Install Node deps
           run: npm ci
           working-directory: frontend
         
         - name: Build Frontend
           run: npm run build
           working-directory: frontend
           env:
             VITE_API_BASE_URL: ${{ secrets.VITE_API_BASE_URL }}

     deploy-backend:
       name: Deploy Backend
       needs: check
       runs-on: ubuntu-latest
       steps:
         - uses: actions/checkout@v4
         - name: Setup Node
           uses: actions/setup-node@v4
           with:
             node-version: '20'
         - name: Install Railway CLI
           run: npm install -g @railway/cli
         - name: Deploy to Railway
           run: railway up
           env:
             RAILWAY_TOKEN: ${{ secrets.RAILWAY_TOKEN }}

     deploy-frontend:
       name: Deploy Frontend
       needs: check
       runs-on: ubuntu-latest
       steps:
         - uses: actions/checkout@v4
         - uses: actions/setup-node@v4
           with:
             node-version: '20'
         - name: Install Vercel CLI
           run: npm install -g vercel
         - name: Deploy to Vercel
           run: vercel --prod --yes
           working-directory: frontend
           env:
             VERCEL_TOKEN: ${{ secrets.VERCEL_TOKEN }}
             VERCEL_ORG_ID: ${{ secrets.VERCEL_ORG_ID }}
             VERCEL_PROJECT_ID: ${{ secrets.VERCEL_PROJECT_ID }}
   ```

2. Nota: Usando `@railway/cli` instalado globalmente via npm. O comando `railway up` faz deploy do diretório atual para o projeto Railway vinculado ao token.

**Acceptance criteria:**
- [ ] Workflow YAML é válido (pode ser validado em https://rhysd.github.io/actionlint/)
- [ ] Trigger: push na branch `main`
- [ ] Job `check` roda `python manage.py check` e `npm run build`
- [ ] Job `deploy-backend` depende de `check`
- [ ] Job `deploy-frontend` depende de `check`
- [ ] Secrets necessários documentados: `RAILWAY_TOKEN`, `VERCEL_TOKEN`, `VERCEL_ORG_ID`, `VERCEL_PROJECT_ID`, `VITE_API_BASE_URL`, `TASK_TRIGGER_SECRET`

### Task 3.3 - Worker e Beat no Railway (Celery) — Fallback para endpoint + cron

**Files modified:** `railway.json`, `Procfile`

**Action:**

**Estratégia primária (Railway free tier):**
Railway free tier suporta apenas 1 processo por serviço. Celery worker + beat não rodam separadamente.

**Fallback implementado:**
1. Endpoint `/api/tasks/trigger/` já implementado no 06-01 (Task 3.4)
2. Usar **cron-job.org** (gratuito, HTTPS, até 60 jobs) para chamar o endpoint semanalmente:
   - Job 1: Segunda 9h → `POST https://<railway-url>/api/tasks/trigger/?secret=<TASK_TRIGGER_SECRET>&task=reminder`
   - Job 2: Segunda 10h → `POST https://<railway-url>/api/tasks/trigger/?secret=<TASK_TRIGGER_SECRET>&task=average`

3. Configurar `TASK_TRIGGER_SECRET` no Railway (mesmo valor do cron-job.org)

**Se fizer upgrade para Railway Pro (pago):**
- Adicionar segundo serviço "worker" no Railway
- Comando: `celery -A backend worker -l info --beat`
- Remover cron-job.org e usar Celery beat nativo

**Acceptance criteria:**
- [ ] Tasks executam semanalmente via cron-job.org (fallback free tier)
- [ ] Endpoint `/api/tasks/trigger/` retorna 200 com resultado da task
- [ ] Secret key protege contra chamadas não autorizadas
- [ ] Documentado: como configurar cron-job.org (URL, schedule, secret)
