<task type="auto">
  <name>Task 1: Atualizar api.js com token storage e interceptador</name>
  <files>frontend-ia/src/config/api.js</files>
  <read_first>frontend-ia/src/config/api.js</read_first>
  <action>
    1. Adicionar funções helpers no topo de `api.js`:
       - `const TOKEN_KEY = 'sa_access_token'`
       - `const REFRESH_KEY = 'sa_refresh_token'`
       - `export function setTokens(access, refresh)` que salva em `localStorage.setItem(TOKEN_KEY, access)` e `localStorage.setItem(REFRESH_KEY, refresh)`
       - `export function getAccessToken()` que retorna `localStorage.getItem(TOKEN_KEY)`
       - `export function getRefreshToken()` que retorna `localStorage.getItem(REFRESH_KEY)`
       - `export function clearTokens()` que remove ambos do localStorage
       - `export function isAuthenticated()` que retorna `!!getAccessToken()`
    2. Alterar `apiRequest` para:
       - Antes do fetch, ler `getAccessToken()`.
       - Se token existir, adicionar header `Authorization: Bearer ${token}` às config.headers.
       - Se response.status === 401, tentar refresh:
         - Chamar `POST /api/auth/refresh/` com `getRefreshToken()` no body.
         - Se refresh retornar novos tokens, salvar via `setTokens` e refazer o request original com novo token.
         - Se refresh falhar, chamar `clearTokens()` e retornar erro.
       - Manter tratamento de erro existente (errorData.erro).
    3. Adicionar endpoint constante: `AUTH_LOGIN: \`${API_BASE_URL}/api/auth/login/\``, `AUTH_REGISTER: \`${API_BASE_URL}/api/auth/register/\``, `AUTH_REFRESH: \`${API_BASE_URL}/api/auth/refresh/\``.
  </action>
