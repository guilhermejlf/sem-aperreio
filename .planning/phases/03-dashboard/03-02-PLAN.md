## Wave 1: Integração com API e Seletor de Período

### Task 2.1 — Adicionar endpoint e função API

**Files modified:** `frontend-ia/src/config/api.js`

**Read first:**
- `frontend-ia/src/config/api.js` (padrão de endpoints existentes)

**Action:**

1. Adicionar ao `API_ENDPOINTS`:
   ```javascript
   DASHBOARD: (mes, ano) => `${API_BASE_URL}/api/dashboard/?mes=${mes}&ano=${ano}`,
   ```

2. Adicionar função helper:
   ```javascript
   export async function fetchDashboard(mes, ano) {
     return await apiRequest(API_ENDPOINTS.DASHBOARD(mes, ano))
   }
   ```

**Acceptance criteria:**
- `api.js` contém `DASHBOARD` endpoint e `fetchDashboard()` function

### Task 2.2 — Refatorar DashboardCharts.vue para consumir API

**Files modified:** `frontend-ia/src/components/DashboardCharts.vue`, `frontend-ia/src/App.vue`

**Read first:**
- `frontend-ia/src/components/DashboardCharts.vue` (código atual — computeds calculam localmente)
- `frontend-ia/src/App.vue` (como o componente é usado: `<DashboardCharts :gastos="gastos" />`)

**Action:**

1. Em `DashboardCharts.vue`:
   - Substituir prop `gastos` por `periodo` (objeto `{mes, ano}`) OU manter `gastos` prop e adicionar fetch interno
   - **Decisão:** Adicionar prop `periodo` (default: mês/ano atual). O componente faz `fetchDashboard(periodo.mes, periodo.ano)` no `mounted` e quando `periodo` muda (watch)
   - Adicionar estado `dashboardData: null` e `loading: false`
   - Computeds (`totalMes`, `mediaDiaria`, `maiorGasto`, `topCategorias`, etc.) leem de `dashboardData` em vez de calcular de `gastos`
   - Remover todas as computeds de agregação local (`gastosDoMes`, `dadosPorCategoria`, `dadosEvolucaoMensal`)
   - Gráfico de evolução (linha) usa `dashboardData.evolucao_12meses` diretamente
   - Gráfico de categorias (pizza) usa `dashboardData.ranking_categorias`

2. Em `App.vue`:
   - `<DashboardCharts>` passa `periodo` em vez de `:gastos="gastos"`
   - Quando período muda, re-fetch do dashboard acontece automaticamente via prop watch no componente

3. Adicionar estado vazio: se `dashboardData.quantidade_gastos === 0`, mostrar ilustração + "Nenhum gasto neste período" + botão para adicionar gasto

**Acceptance criteria:**
- `DashboardCharts.vue` contém `fetchDashboard()` no `mounted`
- Computeds leem de `dashboardData` (não de `gastos`)
- Gráfico de linha usa `evolucao_12meses` do backend
- Gráfico de pizza usa `ranking_categorias` do backend
- Estado vazio renderiza quando não há gastos no período

### Task 2.4 — Adicionar card de comparativo mês anterior

**Files modified:** `frontend-ia/src/components/DashboardCharts.vue`

**Action:**

1. Adicionar 5º card à `stats-grid` (entre "Total do Mês" e "Média Diária"):
   - Ícone: seta (↑ ou ↓)
   - Valor: `{{ formatarVariacao(dashboardData.variacao_percentual) }}`
   - Label: "vs. Mês Anterior"
   - Sub: valor absoluto da diferença
   - Cor: verde quando `variacao_percentual < 0` (gasto diminuiu = bom), vermelho quando `> 0`

2. Adicionar método `formatarVariacao(valor)` que retorna string com sinal e `%`

**Acceptance criteria:**
- Card de comparativo aparece na stats-grid
- Seta ↑ aparece quando variação > 0, ↓ quando < 0
- Cor verde quando diminuiu, vermelho quando aumentou
- Valor formatado como "+12.5%" ou "-8.3%"
